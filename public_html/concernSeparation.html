Separation of Concerns
How the calculator program follows this principle.

Separation of concerns is a software architecture design pattern/principle for separating an application into distinct sections, so each section addresses a separate concern.

  1. Separating program into classes

    2. The entire program is separated into different classes that achieve their own separate goals. For instance, the addition class adds the numbers in a list to give out the values.

  2. Separating tasks into methods

    3. Each task in a class is separated into different methods that achieve the task. For a class like history.calculations, the methods for adding to histry, clearing history, or reading from history are all different and systematically arranged.

  3. Using one method for one task

    4. Since Separation of Concerns is all about maintaining order, this helps us reduce redundancy and create definite method names that make sense as to the task they accomplish.
