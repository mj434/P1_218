OOP(Object Oriented Programming) Glossary
A dictionary of OOP terms

This is a glossary of terms used in Object Oriented Programming.

  1. Object-Oriented Programming

    2. An approach to designing and building applications that are flexible, natural, well-crafted, and testable by focusing on objects that interact cleanly with one another.

  2. Factory

    3. A "Factory" is a class that only contains factory methods

    A "Factory method" is a method inside a class that has an interface (or an abstract class) as a return type and constructs objects implementing this interface based on some internal logic.

  3. Instantiation

    4. The phrase "instantiating a class" means the same thing as "creating an object." When you create an object, you are creating an "instance" of a class, therefore "instantiating" a class.

  4. Object

    5. An instance of a class that is created at runtime.

  5. Class

    6. The code describing a particular entity in the application, such as Customer, Order, or Address. The class contains the code defining the properties and methods.

  6. Namespace

    7. A namespace is a system that has a unique name for each and every object in Python. An object might be a variable or a method.

  7. Constructor

    8. Code that is executed each time an instance of the class is created.

  8. Fixture

    9. Fixtures are functions, which will run before each test function to which it is applied.

  9. Type hint

    10. Type hinting is a formal solution to statically indicate the type of a value within your Python code.

  10. Type cast

    11. Typecasting, or type conversion, is a method of changing an entity from one data type to another.

  11. Unit test

    12. A unit test is a way of testing a unit - the smallest piece of code that can be logically isolated in a system. In most programming languages, that is a function, a subroutine, a method or property.

  12. Static

    13. In object-oriented programming, there is also the concept of a static member variable, which is a "class variable" of a statically defined class, i.e., a member variable of a given class which is shared across all instances (objects), and is accessible as a member variable of these objects.

  13. Class Method

    14. A class method is a method which is bound to the class and not the object of the class. They have the access to the state of the class as it takes a class parameter that points to the class and not the object instance.

  14. Facade

    15. A facade is an object that serves as a front-facing interface masking more complex underlying or structural code.
